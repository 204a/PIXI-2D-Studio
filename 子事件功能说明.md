# 子事件功能使用说明

## 功能概述

子事件功能已完整实现，允许在事件内嵌套子事件，实现更复杂的游戏逻辑。

## 主要特性

### 1. 嵌套结构
- 支持最多2层嵌套（主事件 → 子事件 → 孙事件）
- 子事件只在父事件条件满足时执行
- 子事件可以有自己独立的条件和动作

### 2. 视觉区分
- **主事件**: 绿色边框 (#4CAF50)，深灰背景 (#2a2a2a)
- **子事件**: 蓝色边框 (#2196F3)，更深背景 (#232323)
- 子事件显示缩进 (20px) 和树形连接符 (└─)

### 3. 执行逻辑
```
主事件
  ├─ 检查主事件条件
  ├─ 执行主事件动作
  └─ 如果条件满足 → 执行所有子事件
       ├─ 子事件1
       │   ├─ 检查子事件条件
       │   └─ 执行子事件动作
       └─ 子事件2
           ├─ 检查子事件条件
           └─ 执行子事件动作
```

## 使用方法

### 创建子事件
1. 选择任意对象
2. 添加一个主事件
3. 点击主事件上的 **"➕子事件"** 按钮
4. 选择子事件类型（游戏开始时/每帧更新时）
5. 为子事件添加条件和动作

### 删除事件
- 删除主事件会**自动删除**所有子事件
- 删除子事件不影响父事件
- 支持递归删除（删除有子事件的事件）

### 限制
- 最多2层嵌套
- 达到2层时，不再显示"➕子事件"按钮

## 应用示例

### 示例1: 条件组合控制
```
主事件: 每帧更新时
  条件: 按键按住 "空格"
  动作: (无)
  
  子事件1:
    条件: X坐标 < 400
    动作: 移动 (5, 0)
  
  子事件2:
    条件: X坐标 >= 400
    动作: 移动 (-5, 0)
```
效果: 按住空格时，根据位置不同向不同方向移动

### 示例2: 连续触发
```
主事件: 每帧更新时
  条件: 碰撞检测 "coin"
  动作: 淡出
  
  子事件:
    条件: 透明度 < 0.1
    动作: 设置位置 (1000, 1000)
```
效果: 碰到金币后淡出，完全透明后移出屏幕

### 示例3: 双人游戏分支
```
主事件: 每帧更新时
  条件: (无)
  动作: (无)
  
  子事件1: 玩家1控制
    条件: 按键按住 "w"
    动作: 移动 (0, -5)
  
  子事件2: 玩家2控制
    条件: 按键按住 "↑"
    动作: 移动 (0, -5)
```

## 技术实现

### BehaviorSystem 新增方法
- `addSubEvent(parentId, objectId, eventType, actions, options)`: 添加子事件
- `removeBehavior(behaviorId)`: 递归删除，包含所有子事件

### 数据结构
```javascript
{
  id: "behavior_123",
  objectId: "obj_456",
  eventType: "update",
  conditions: [...],
  actions: [...],
  subEvents: ["behavior_789", "behavior_012"],  // 子事件ID列表
  parentId: null  // 父事件ID（主事件为null）
}
```

### EventEditorUI 新增方法
- `buildEventTree(behavior, allBehaviors)`: 递归构建事件树
- `addSubEvent(parentId)`: 显示子事件添加对话框

## 后续优化方向

1. **拖拽重排**: 支持在不同层级间拖拽事件
2. **折叠展开**: 主事件可折叠隐藏子事件
3. **复制粘贴**: 复制整个事件树（包括子事件）
4. **模板库**: 预设常用的事件组合模板
5. **可视化连线**: 用连线显示父子关系
6. **条件OR逻辑**: 目前只支持AND，可添加OR分支
